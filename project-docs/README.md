## 🚀 快速开始

### 第一步: 阅读总览文档

请先阅读 **`00-项目开发总览.md`**,了解:
- 项目背景和核心价值
- 技术栈和开发工具
- 完整的文档导航
- 开发路线图和里程碑
- 可行性结论和风险评估
- 开发任务分配

### 第二步: 查看可行性分析

阅读 **`01-需求可行性分析报告.md`**,了解:
- HDty 框架组件支撑度评估
- 核心功能技术方案可行性
- 风险评估和应对方案
- 工作量估算

### 第三步: 了解前端开发计划

阅读 **`02-前端开发计划.md`**,了解:
- 前端目录结构规划
- 核心功能实现方案(统计列表、动态表单、明细弹窗)
- 开发任务拆分和优先级
- 关键技术点(hd-build-table、hd-form 使用)
- HDty 框架组件清单

### 第四步: 开始开发

根据开发计划,按照以下顺序进行:

```
1. 技术预研 (1天)
   └─ 确认 hd-build-table 和 hd-form 支持度

2. 数据库设计 (2天)
   └─ 创建数据库初始化脚本

3. 基础设施搭建 (2天)
   ├─ 前端 API 封装
   ├─ Vuex 状态管理
   └─ 后端项目框架

4. 核心功能开发 (3-4天)
   ├─ 通用组件(动态表单、字段渲染器)
   ├─ 统计列表页面
   └─ 明细弹窗组件

5. 配置管理开发 (2-3天)
   ├─ 列配置管理
   └─ 表单配置管理

6. 联调测试 (2天)

7. 性能优化与文档完善 (2天)
```

## 🔍 常见问题

### Q1: 为什么要单独创建 project-docs 目录?

**A**: 
- `dev-md/` 目录存放的是**需求与设计文档**(业务需求、技术架构、数据库设计等),面向产品、架构师、设计人员
- `project-docs/` 目录存放的是**开发执行文档**(可行性分析、开发计划、任务分配、开发日志等),面向开发人员

两者职责不同,分开管理更清晰。

### Q2: 已有 dev-md/前端开发文档.md,为什么还要 project-docs/02-前端开发计划.md?

**A**:
- `dev-md/前端开发文档.md`: 偏向于**技术方案和代码示例**,讲解如何实现多级表头、动态表单等技术点
- `project-docs/02-前端开发计划.md`: 偏向于**任务拆分和开发计划**,列出具体要做哪些任务、优先级、工作量估算

一个是"怎么做",一个是"做什么"。

### Q3: 如何使用这些文档?

**A**:
1. **项目启动前**: 阅读 `00-项目开发总览.md` 和 `01-需求可行性分析报告.md`,了解项目全貌
2. **技术预研**: 查看 `02-前端开发计划.md` 的"关键技术点"部分,确认技术方案
3. **开发过程中**: 按照开发计划执行,记录问题到 `06-开发日志.md`
4. **测试阶段**: 参考 `05-测试计划.md` 编写测试用例
5. **项目交付**: 整理所有文档,形成完整的项目文档
